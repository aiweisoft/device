# 设备管家 - 医疗设备全生命周期管理

基于 uni-starter v2.2.15 模板开发的医疗设备管理移动端 App，支持 H5/App/小程序多端运行。

## 项目定位

与 [neihao-web](https://github.com/aiweisoft/neihao-web) 管理后台共享同一 uniCloud 支付宝云环境，构成完整的医疗设备管理系统：

| 项目 | 定位 | 用户 |
|------|------|------|
| **device**（本仓库） | 移动端 App（H5/App/小程序） | 普通用户/医护人员 |
| neihao-web | 管理后台 Web 端 | 管理员 |

## 功能模块

- **首页仪表盘**：设备统计卡片、快捷操作入口、待处理提醒
- **设备管理**：设备列表（搜索/分类/状态标签）、设备详情、扫码
- **报修与维修**：申请报修、报修记录、维修记录、维修详情
- **提醒消息**：已读/未读标记、类型标签
- **用户中心**：登录注册、个人设置、关于

## 技术栈

- 框架：uni-app（Vue2/Vue3 兼容）
- 后端：uniCloud 支付宝云（clientDB + 云对象）
- 认证：uni-id-pages 用户认证
- 推送：UniPush
- 国际化：VueI18n
- UI：uni-ui 组件库

## 用户角色与权限

系统支持两种用户角色，前端根据角色展示不同的功能入口：

| 角色 | role_id | 分配方式 | 可见功能 |
|------|---------|---------|---------|
| **报修用户** | `repair_user` | 新用户注册自动分配 | 提交报修 + 查看自己的报修记录 |
| **维修用户** | `app_maintainer` | 管理员在 neihao-web 后台手动分配 | 查看所有报修单 + 创建/管理维修记录 |

### 角色升级

管理员在 neihao-web 管理后台的"用户管理"中将目标用户的 `role` 字段添加 `app_maintainer` 即可升级为维修用户。角色变更后用户需重新登录以刷新前端权限。

### 权限校验

| 操作 | 后端校验 |
|------|---------|
| 创建维修记录 | `auth.role` 须包含 `app_maintainer` |
| 创建报修单 | 任何登录用户 |
| 前端 UI | 根据 `store.userInfo.role` 动态渲染 |

## 报修与维修操作流程

### 首页入口

根据用户角色，首页快捷操作不同：

**报修用户：**

| 快捷操作 | 跳转页面 | 说明 |
|----------|---------|------|
| 设备列表 | `pages/device/device-list` | Tab 切换 |
| 我的报修 | `pages/repair-request/repair` | 双 Tab（申请报修 + 报修记录） |
| 保养提醒 | `pages/alert/list` | 提醒列表 |

**维修用户：**

| 快捷操作 | 跳转页面 | 说明 |
|----------|---------|------|
| 设备列表 | `pages/device/device-list` | Tab 切换 |
| 所有报修 | `pages/repair-request/list` | 查看所有用户的报修单 |
| 维修记录 | `pages/repair-request/list?tab=1` | 查看所有维修记录 |
| 保养提醒 | `pages/alert/list` | 提醒列表 |

### 申请报修（报修用户）

1. 点击首页 **"我的报修"** → 进入 `repair.vue` 的 **"申请报修"** Tab
2. 表单自动填入当前登录用户的**报修人**和**联系电话**
3. 选择故障设备（仅显示使用中的设备）、报修日期、故障描述、紧急程度
4. 提交后：创建报修单（状态=待处理），同时将设备状态置为**维修中**（status=3）
5. 提交成功自动切换到 **"报修记录"** Tab 查看最新记录

### 报修记录

- **报修用户**：在 `repair.vue` **"报修记录"** Tab 中查看自己的所有报修单
- **维修用户**：在 `list.vue` **"所有报修"** Tab 中查看全部用户的报修单
- 每个报修单展示设备名称、故障描述、紧急程度、处理状态标签
- 点击卡片进入**报修详情页**（`detail.vue`）
- **编辑**：修改报修信息（仅限未关联维修记录的报修单）
- **删除**：删除报修单（存在关联维修记录时禁止删除），同时恢复设备状态为使用中

### 维修记录

- **报修用户**：显示与自己报修单关联的维修记录
- **维修用户**：显示所有维修记录，并可点击"去维修"/"记录维修"创建新维修记录
- 点击卡片进入**维修详情页**（`repair-detail.vue`）
- 支持编辑和删除维修记录

### 操作按钮

**`list.vue`"所有报修/我的报修"Tab：**

| 按钮 | 出现条件 | 适用角色 | 功能 |
|------|---------|---------|------|
| 去维修 | 状态=待处理 | 报修用户 / 维修用户 | 跳转 `repair-add.vue` 创建维修记录 |
| 记录维修 | 状态=处理中 | 报修用户 / 维修用户 | 跳转 `repair-add.vue` 继续记录维修 |
| 编辑 | 无关联维修记录 | 维修用户 | 编辑报修单信息 |
| 删除 | — | 维修用户 | 删除报修单 |

### 设备状态联动

| 操作 | 设备状态变化 |
|------|-------------|
| 提交报修 | status: 2(使用中) → 3(维修中) |
| 删除报修单 | status: 3(维修中) → 2(使用中) |
| 维修完成（结果=已修复） | status: 3(维修中) → 2(使用中)，报修单→已完成 |
| 删除维修记录 | status: 2(使用中) → 3(维修中)，报修单→恢复待处理 |

### 数据表

| 表名 | 说明 |
|------|------|
| `medical-device-repair-request` | 报修单（软删除，含 creator、status、urgency、device_id） |
| `medical-device-repair` | 维修记录（软删除，关联报修单、含 result、cost、repair_person 等） |
| `medical-device` | 设备台账（状态联动更新，status 字段：2=使用中/3=维修中） |

## 快速开始

1. 使用 HBuilderX 打开项目
2. 在 `manifest.json` 中配置各平台 AppID
3. 右键 `uniCloud-alipay/database/db_init.json` → 初始化云数据库
4. 运行到浏览器 / 手机 / 小程序模拟器
