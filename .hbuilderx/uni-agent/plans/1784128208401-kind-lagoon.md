# 报修用户 & 维修用户 — 当前任务计划

## 本期任务：ucenter.vue 菜单项角色化调整

### 目标

修改 `pages/ucenter/ucenter.vue` 的个人中心菜单，将原来的单一"我的报修"入口拆分为"我的报修"和"维修记录"两项，并加入与首页快捷操作一致的角色校验逻辑。

### 需求描述

- 菜单显示两个条目："我的报修"和"维修记录"（始终显示，不按角色隐藏）
- 点击"我的报修"：若为维修用户 → toast "仅报修用户可用"；普通/报修用户 → 跳转 `/pages/repair-request/repair`
- 点击"维修记录"：若非维修用户 → toast "仅维修用户可用"；维修用户 → 跳转 `/pages/repair-request/list?tab=1`
- 与首页 index.vue 的 `execAction` 行为完全一致

### 实现方案

#### 1. 添加 `isMaintainer` 计算属性

```javascript
import { store } from '@/uni_modules/uni-id-pages/common/store.js';

computed: {
    isMaintainer() {
        return store.hasLogin && store.userInfo.role?.includes('app_maintainer');
    }
}
```

#### 2. 修改菜单数据结构

将 ucenterList 第一个子数组的"我的报修"替换为两个条目，使用 `event` 属性（而非 `to`）来拦截点击：

```javascript
ucenterList: [
    [{
        "title": "我的报修",
        "to": "",           // 清空，由 event 控制导航
        "icon": "paperplane",
        "event": "toMyRepair"
    }, {
        "title": "维修记录",
        "to": "",           // 清空，由 event 控制导航
        "icon": "search",
        "event": "toRepairRecords"
    }, {
        "title": "提醒消息",
        "to": "/pages/alert/list",
        "icon": "notification"
    }, {
        "title": "设备列表",
        "to": "",
        "event": "toDeviceList",
        "icon": "list"
    }],
    // ... 其余不变
]
```

#### 3. 修改 `ucenterListClick` 方法

```javascript
ucenterListClick(item) {
    if (item.event) {
        if (item.event === 'toMyRepair') {
            if (this.isMaintainer) {
                uni.showToast({ title: '仅报修用户可用', icon: 'none' });
                return;
            }
            uni.navigateTo({ url: '/pages/repair-request/repair' });
            return;
        }
        if (item.event === 'toRepairRecords') {
            if (!this.isMaintainer) {
                uni.showToast({ title: '仅维修用户可用', icon: 'none' });
                return;
            }
            uni.navigateTo({ url: '/pages/repair-request/list?tab=1' });
            return;
        }
        // 其他 event（如 toDeviceList）
        this[item.event]();
    }
}
```
