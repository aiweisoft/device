# 报修用户 & 维修用户 — 角色权限体系实现计划

## 概述

实现两种用户角色的权限控制：
- **报修用户（repair_user）**：提交报修 + 查看自己的报修记录 + 查看关联的维修记录
- **维修用户（app_maintainer）**：查看所有报修单 + 创建/管理维修记录

新用户注册默认分配为报修用户，管理员在 neihao-web 后台手动升级为维修用户。

---

## 一、角色定义与数据初始化

### 1.1 新增 `uni-id-roles` 角色记录

在 `uniCloud-alipay/database/uni-id-roles.init_data.json` 中新增 `repair_user` 角色：

```json
[
  { "role_id": "admin", "role_name": "超级管理员", "permission": [], "comment": "" },
  { "role_id": "repair_user", "role_name": "报修用户", "permission": [], "comment": "默认角色，新用户注册自动分配" }
]
```

`app_maintainer` 角色由 neihao-web 管理后台创建（或手动在数据库中添加）。

### 1.2 配置新用户默认角色

修改 `uni_modules/uni-config-center/uniCloud/cloudfunctions/common/uni-config-center/uni-id/config.json`：

```json
"userRegisterDefaultRole": ["repair_user"]
```

这样 uni-id-co 的 `register.js` 在用户注册时会自动将 `repair_user` 注入 `user.role` 数组。

## 二、前端 Store 扩展

### 2.1 获取用户角色

修改 `uni_modules/uni-id-pages/common/store.js` 的 `updateUserInfo()`：

```javascript
// 原代码
let res = await usersTable.where("'_id' == $cloudEnv_uid")
    .field('mobile,nickname,username,email,avatar_file')
    .get()

// 改为
let res = await usersTable.where("'_id' == $cloudEnv_uid")
    .field('mobile,nickname,username,email,avatar_file,role')
    .get()
```

`userInfo.role` 现在会是一个字符串数组，如 `["repair_user"]` 或 `["app_maintainer"]`。

### 2.2 新增角色判断 computed（在 store.js 中）

```javascript
hasRole(name) {
    return (this.userInfo.role || []).includes(name)
}
```

或各页面直接通过 `store.userInfo.role?.includes('app_maintainer')` 判断。

## 三、Schema 权限调整

### 3.1 维修记录（medical-device-repair）

```json
"create": "'app_maintainer' in auth.role || 'app_admin' in auth.role",
"update": "'app_maintainer' in auth.role || 'app_admin' in auth.role || auth.uid != null",
"delete": "'app_admin' in auth.role || auth.uid != null"
```

**关键变更**：`create` 移除 `auth.uid != null`，仅允许维修用户创建维修记录。`update` 保留 `auth.uid != null`（编辑自己的维修记录）。

### 3.2 报修单（medical-device-repair-request）

```json
"create": "auth.uid != null",
"update": "auth.uid != null",
"delete": "auth.uid != null"
```

保持现有权限不变，所有登录用户都可提交报修。

### 3.3 设备台账（medical-device）

```json
"update": "'UPDATE_MEDICAL_DEVICE' in auth.permission || auth.uid != null"
```

保持现有权限（报修时需更新设备 status=3）。

## 四、页面级角色控制

### 4.1 首页 index.vue — 动态快捷操作

将 `actions` 从 `data` 改为 `computed`，根据角色动态生成：

| 快捷操作 | 报修用户 | 维修用户 |
|---------|:---:|:---:|
| 设备列表 | ✅ | ✅ |
| 我的报修 → repair.vue | ✅ | ❌ |
| 所有报修 → list.vue?tab=0 | ❌ | ✅ |
| 维修记录 → list.vue?tab=1 | ❌ | ✅ |
| 保养提醒 | ✅ | ✅ |

```javascript
computed: {
    actions() {
        const base = [
            { label: '设备列表', icon: 'list', bgColor: '#10b981', action: 'toDeviceList' }
        ];
        if (store.hasLogin) {
            if (store.userInfo.role?.includes('app_maintainer')) {
                // 维修用户
                base.push(
                    { label: '所有报修', icon: 'bars', bgColor: '#06b6d4', action: 'toAllRequests' },
                    { label: '维修记录', icon: 'search', bgColor: '#8b5cf6', action: 'toRepairRecords' }
                );
            } else {
                // 报修用户（默认）
                base.push(
                    { label: '我的报修', icon: 'paperplane', bgColor: '#f59e0b', action: 'toMyRepair' }
                );
            }
            base.push(
                { label: '保养提醒', icon: 'notification', bgColor: '#ef4444', action: 'toAlerts' }
            );
        }
        return base;
    }
}
```

### 4.2 list.vue — 动态 Tab 与查询条件

**Tab 标签**：改为 `computed`，根据角色动态生成：

```javascript
computed: {
    tabs() {
        if (store.userInfo.role?.includes('app_maintainer')) {
            return ['所有报修', '维修记录'];
        }
        return ['我的报修', '维修记录'];
    }
}
```

**buildWhere**：根据角色调整过滤条件：

```javascript
buildWhere(tab) {
    const uid = this.uid;
    const isMaintainer = store.userInfo.role?.includes('app_maintainer');
    if (tab === 0) {
        const w = { deleted: 0 };
        if (!isMaintainer && uid) w.creator = uid;  // 报修用户只看自己的
        if (this.deviceId) w.device_id = this.deviceId;
        this.orderby = 'request_date desc';
        this.where = w;
    } else {
        this.orderby = 'repair_date desc';
        if (isMaintainer) {
            this.where = { deleted: 0 };  // 维修用户看全部维修记录
        } else {
            const ids = this.myRequestIds;
            this.where = ids.length
                ? { deleted: 0, repair_request_id: dbCmd.in(ids) }
                : { deleted: 0, _id: '__nonexistent__' };
        }
    }
}
```

**操作按钮**：维修用户的 tab 0（所有报修）同样显示"去维修"/"记录维修"按钮。

### 4.3 repair.vue — 报修用户专属页面

维修用户不应使用此页面（由首页快捷操作控制，报修用户走 repair.vue，维修用户走 list.vue）。

但需防御性处理：若维修用户意外进入，做如下调整：

```javascript
computed: {
    tabs() {
        if (store.userInfo.role?.includes('app_maintainer')) {
            return ['报修记录'];  // 去掉申请报修Tab
        }
        return ['申请报修', '我的报修'];
    }
}
```

### 4.4 repari-add.vue — 维修记录表单

前端不做拦截（由 schema 权限控制）。维修用户提交维修记录时，`creator` 字段应设置为当前用户。

在 submit 新增模式下增加 `creator` 字段：

```javascript
await db.collection('medical-device-repair').add({
    ...repairData,
    deleted: 0,
    creator: store.userInfo?._id || ''
});
```

## 六、实施顺序

1. **数据库层**：修改 schema 权限 + 添加角色记录 + 配置默认角色
2. **Store 层**：`updateUserInfo` 增加 role 查询
3. **UI 层**：逐页面添加角色判断逻辑（index → list → repair → repair-add）
4. **验证**：注册新用户 → 确认自动获取 `repair_user` 角色 → 测试功能限制
