# 提交设备维修逻辑分析报告

## 一、概述

"提交设备维修"功能是一个**两阶段业务流**：
1. **阶段一（提交报修）**：用户填写报修表单 → 调用云对象原子化写入报修单 + 更新设备状态
2. **阶段二（记录维修）**：维修人员处理完成后，写入维修记录 → 联动更新报修单状态 + 恢复设备状态

### 3.2 报修表单页 (`add.vue`)

**表单字段**：
| 字段 | 组件 | 必填 | 说明 |
|------|------|------|------|
| 关联设备 | `uni-combox`（搜索选择） | ✅ | 从 `medical-device` 表加载"使用中(status=2)且未删除"的设备 |
| 报修日期 | `uni-datetime-picker` | ❌ | 默认当前时间 |
| 报修人 | `uni-easyinput` | ❌ | 文本输入 |
| 联系电话 | `uni-easyinput` (number) | 格式校验 | 正则 `/^1\d{10}$/` |
| 故障描述 | `uni-easyinput` (textarea) | ✅ | 必填 |
| 紧急程度 | `uni-data-select` | ❌ | 1=一般 / 2=紧急 / 3=非常紧急（默认1） |
| 备注 | `uni-easyinput` (textarea) | ❌ | 选填 |

**表单验证规则** (`add.vue:55-59`)：
```javascript
rules: {
    device_id: { rules: [{ required: true, errorMessage: '请选择设备' }] },
    fault_description: { rules: [{ required: true, errorMessage: '请描述故障现象' }] },
    phone: { rules: [{ pattern: /^1\d{10}$/, errorMessage: '请输入正确的手机号' }] }
}
```

**设备选择逻辑** (`add.vue:69-83`)：
- 加载 `medical-device` 表中 `deleted == 0 && status == 2`（使用中）的设备
- 仅查 `_id, name, code` 三个字段，`limit(10000)`
- 构造候选列表格式：`"设备名称 (编号)"`，通过 `deviceOptionMap` 映射回 `_id`
- 若从详情页带参数跳转（`onLoad(e)`），自动预填设备

**核心提交方法** (`add.vue:107-126`)：
```javascript
async submit() {
    this.$refs.form.validate().then(async () => {
        this.submitting = true
        try {
            const repairCo = uniCloud.importObject('medical-device-repair-co')
            const res = await repairCo.submitRepair(this.formData)
            if (res.code === 0) {
                uni.showToast({ title: res.message, icon: 'success' })
                setTimeout(() => uni.navigateBack(), 1500)
            } else {
                uni.showToast({ title: res.message, icon: 'none' })
            }
        } catch (e) {
            uni.showToast({ title: e.message || '提交失败', icon: 'none' })
        } finally { this.submitting = false }
    })
}
```

### 3.3 云对象 `medical-device-repair-co` (`index.obj.js`)

**前置鉴权** (`_before:`, 第4-10行)：
```javascript
_before: function() {
    const uid = this.getClientInfo().uid
    if (!uid) throw new Error('请先登录')
    this.uid = uid
}
```

**`submitRepair()` 方法完整逻辑** (第12-57行)：

```
输入: { device_id, request_date, requester, phone, fault_description, urgency, remark }

步骤1: 参数校验
  ✅ device_id 和 fault_description 不能为空
  ❌ → 返回 { code: 1, message: '设备和故障描述不能为空' }

步骤2: 设备存在性校验
  查询 medical-device.doc(device_id)，取 status + name
  ❌ 设备不存在 → 返回 { code: 1, message: '设备不存在' }

步骤3: 防重复报修校验
  检查 device.status === 3（维修中）
  ❌ 已在维修中 → 返回 { code: 1, message: '该设备已在维修中，无需重复报修' }

步骤4: 保存设备原状态
  const previousStatus = device.status  // 如 1(正常) 或 2(使用中)

步骤5: 新增报修单
  db.collection('medical-device-repair-request').add({
      device_id, request_date, requester, phone,
      fault_description, urgency, remark,
      status: 1,        // 待处理
      deleted: 0,
      creator: this.uid,
      created_at: now
  })

步骤6: 更新设备状态
  db.collection('medical-device').doc(device_id).update({
      previous_status: previousStatus,  // 报修前状态（用于恢复）
      status: 3,                        // 3=维修中
      updated_at: now,
      updater: this.uid
  })

输出: { code: 0, message: '报修提交成功', data: { id, previous_status } }
```

### 3.4 数据表变化

报修提交后，两张表的变化：

**`medical-device-repair-request` 表** — INSERT：
| 字段 | 值 | 说明 |
|------|-----|------|
| `status` | `1` | 待处理 |
| `deleted` | `0` | 未删除 |
| `creator` | `this.uid` | 当前登录用户ID |

**`medical-device` 表** — UPDATE：
| 字段 | 值 | 说明 |
|------|-----|------|
| `status` | `3` | 维修中 |
| `previous_status` | 原状态值 | 报修前状态，用于维修后恢复 |

### 4.2 `repair-add.vue` 提交逻辑

**表单字段**：维修结果(1=已修复/2=部分修复/3=无法修复)、维修日期、故障原因、维修方式、维修人员、维修公司、维修费用、备注

**核心提交逻辑**（三步前端 clientDB 操作）：

```javascript
async submit() {
    // 步骤1: 新增维修记录
    await db.collection('medical-device-repair').add({
        device_id, repair_request_id: this.requestId,
        repair_date, result, fault_reason, repair_method,
        repair_person, repair_company, cost, remark, deleted: 0
    })

    // 步骤2+3: 根据维修结果更新报修单和设备状态
    if (result === 1) {
        // 已修复：
        //   - 报修单 status → 3（已完成）
        //   - 设备 status → previous_status（恢复原状态），清除 previous_status
        requestUpdate.status = 3
        deviceUpdate.status = previousStatus
        deviceUpdate.previous_status = dbCmd.remove()
    } else {
        // 部分修复/无法修复：
        //   - 报修单 status → 2（处理中）
        //   - 设备保持 status=3（维修中）
        requestUpdate.status = 2
    }

    await Promise.all([
        db.collection('medical-device-repair-request').doc(requestId).update(requestUpdate),
        db.collection('medical-device').doc(deviceId).update(deviceUpdate)
    ])
}
```

## 五、状态流转图

### 设备状态 (`medical-device.status`)

```
status=X (正常/使用中等)         ← 报修前
    │  submitRepair() 保存 previous_status=X
    ▼
status=3 (维修中)                ← 报修后
    │  repair-add 提交维修结果
    ├─ result=1(已修复) ──► status=X (恢复原状态) ← 清除 previous_status
    └─ result≠1 ──────────► status=3 (保持维修中)
```

### 报修单状态 (`medical-device-repair-request.status`)

```
status=1 (待处理)                ← 提交后初始状态
    │  repair-add 提交维修结果
    ├─ result=1(已修复) ──► status=3 (已完成)
    └─ result≠1 ──────────► status=2 (处理中)
```

## 六、权限控制

| 操作 | 权限要求 |
|------|----------|
| 创建报修单 | **任意登录用户** (`auth.uid != null`) |
| 创建维修记录 | `app_maintainer` 角色 或 `app_admin` 角色 或 `CREATE_MEDICAL_DEVICE_REPAIR` 权限 |
| 更新设备状态 | `UPDATE_MEDICAL_DEVICE` 权限 |
| 页面访问 | `uniIdRouter.needLogin` 配置了三个 repair-request 页面需登录 |

**关键点**：报修申请是唯一允许普通登录用户创建的业务记录，体现了"任何用户都可以提交报修"的设计。但维修记录的创建需要维修员或管理员角色。

## 七、防重复报修机制

在云对象 `submitRepair()` 中（第24-26行）：
```javascript
if (device.status === 3) {
    return { code: 1, message: '该设备已在维修中，无需重复报修' }
}
```
- 报修提交前检查设备当前状态
- 若 `status === 3`（维修中），直接拒绝
- 前端 `add.vue` 加载设备列表时也只查 `status == 2`（使用中）的设备，进一步减少误操作

---

## 八、设计总结

1. **两阶段松耦合**：报修和维修是两个独立页面，通过 `repair_request_id` 关联
2. **原子性保证**：阶段一通过云对象保证"新增报修单 + 更新设备状态"的原子性；阶段二通过前端 `Promise.all` 并行更新
3. **设备状态保护**：`previous_status` 字段实现报修期间的"状态暂存"与维修完成的"自动恢复"
4. **多入口设计**：首页、设备详情页、个人中心均可触发，提升用户体验
5. **鉴权分级**：普通用户可提交报修，维修记录需要维护员/管理员权限

## 验证方法

如需验证此功能，可通过以下步骤：

1. 以登录用户身份访问首页 → 点击「提交报修」快捷操作
2. 填写报修表单（选择设备、填写故障描述）
3. 提交后检查：
   - `medical-device-repair-request` 表新增一条 `status=1` 的记录
   - 对应设备 `status` 变为 `3`，`previous_status` 保存原状态
4. 进入「我的报修」列表 → 点击「记录维修」
5. 填写维修结果（选择"已修复"）
6. 检查：
   - `medical-device-repair` 表新增一条维修记录
   - 报修单 `status` 变为 `3`（已完成）
   - 设备 `status` 恢复原状态，`previous_status` 被清除
